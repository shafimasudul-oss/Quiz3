The sortColors problem gives us three different numbers: 0, 1, and 2. These numbers represent the colors and they're given to us in an array. Our approach to the problem was to do a count sort. We first initialized three different counters for each number (zerocount, onecount, twocount). Then we iterated through the array and counted how many instances of each integer appeared. After we had that, we changed the original nums array. We started at index 0 and began replacing the numbers in the original array (i.e, if original array was [1, 0], it would rewrite the "1" to a 0), and then we did that for each number in order of zero, one, two. This wouldn't change the size of the array since all the counts add up to the size of the array.

The maximumGap problem gives us an unsorted array of integers, and we need to find the maximum difference between two consecutive numbers after the array is sorted. We use a bucket sort idea to solve it in linear time. Our approach is to first find the minimum and maximum values in the array. Then we divide the range between them into evenly sized buckets, where each bucket represents a range of values. Instead of storing all elements, each bucket only keeps track of the minimum and maximum value that fall into it. After distributing all numbers into their corresponding buckets, we scan through the buckets in order. We skip empty buckets and compute the difference between the minimum value of the current non-empty bucket and the maximum value of the previous non-empty bucket. This difference represents a possible gap. We keep track of the largest such gap while iterating. Since the maximum gap must occur between elements in different buckets (not within the same bucket), this method ensures we find the correct answer in O(n) time without fully sorting the array.
